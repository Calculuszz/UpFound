# UpFound — Product Spec

> **One-liner:** ระบบตามหาของหายในพื้นที่ปิด (ห้าง/สนามบิน/อาคาร) ที่ใช้ AI สร้าง "journey ของวัตถุ" จากกล้องวงจรปิด แล้วให้เจ้าของพิสูจน์ตัวตนด้วยสิ่งที่กล้องเห็นจริง — privacy-preserving และไม่เดาเกินกว่าหลักฐานที่มี

สถานะ: เดโม / ประกวด (Demo Day 16 ก.ค. 2026)

---

## 1. ปัญหา

ของหายในพื้นที่ปิดเป็นปัญหาที่แก้ด้วยแรงคนล้วนๆ — เจ้าหน้าที่ต้องไล่ดูกล้องเอง, ของกองที่เคาน์เตอร์โดยไม่มีระบบจับคู่กับเจ้าของ, คนหาของต้องเดินถามทีละจุด อัตราการได้ของคืน (recovery rate) ต่ำ และกระบวนการพิสูจน์ความเป็นเจ้าของก็หละหลวม (ใครบอกว่าเป็นของตัวเองก็ได้)

## 2. ผู้ใช้และบริบท

| ผู้ใช้                            | ต้องการ                                                                         |
| --------------------------------- | ------------------------------------------------------------------------------- |
| **ผู้ที่ทำของหาย**                | แจ้งง่าย, รู้เร็วว่าของอยู่ไหน, รับคืนสะดวก                                     |
| **เจ้าหน้าที่ / รปภ.**            | เครื่องมือช่วยจับคู่ของกับเจ้าของ, ลดงานไล่ดูกล้อง, ตัดสินใจปล่อยของอย่างมั่นใจ |
| **เจ้าของพื้นที่ (ห้าง/สนามบิน)** | recovery rate สูงขึ้น, ไม่ละเมิด PDPA, ภาพลักษณ์บริการดี                        |

## 3. หลักการหลัก (Core Principle)

**ไม่ให้ AI เดาว่าใครเป็นเจ้าของ** แต่ให้ระบบ **สร้าง journey ของวัตถุที่ตรวจสอบได้** แล้วให้คน **ยืนยันด้วยสิ่งที่ระบบเช็คกับฟุตเทจได้จริง** — AI เป็นผู้ช่วยเจ้าหน้าที่ตัดสินใจ ไม่ใช่ผู้ตัดสินแทน

หลักนี้คือสิ่งที่แยก UpFound ออกจากการเป็นแค่ "search engine บนกล้อง" และเป็นตัวปิดช่องโหว่ด้านความน่าเชื่อถือ/privacy พร้อมกัน

## 4. User Journey

**เส้นตรวจจับ (ทำงานเงียบๆ):** กล้องเห็นของถูกวางนิ่งเกินเวลา → ระบบจับเป็น "ของถูกทิ้ง" (advisory) → เก็บ journey ของวัตถุไว้รอเจ้าของ

**เส้นตามหา (เริ่มเมื่อมีคนแจ้ง):**

1. ผู้ใช้แจ้งของหาย — กรอกประเภท/สี/ยี่ห้อ/จุดเด่น + โซน + ช่วงเวลา (รูปไม่บังคับ)
2. ระบบหา candidate ให้เกือบทันที พร้อมเฟรมจริง + timeline
3. ผู้ใช้เลือกอันที่ใช่ จากภาพในกล้อง
4. พิสูจน์ตัวตนด้วยคำถามที่กล้องเห็นได้ (นั่งโซนไหน / มากี่คน / เสื้อสีอะไร / เดินมาทางไหน)
5. ผ่าน → ได้ QR claim pass
6. ไปจุดรับของ เจ้าหน้าที่เปิดของยืนยันของข้างใน → รับคืน

## 5. ฟีเจอร์ & ลำดับความสำคัญ

| ฟีเจอร์                                  | Priority  | หมายเหตุ                           |
| ---------------------------------------- | --------- | ---------------------------------- |
| ฟอร์มแจ้งของหายแบบ structured            | P0        | รูป optional                       |
| จับคู่ candidate + แสดงเฟรม/timeline     | P0        | จุดขายหลัก                         |
| confidence + abstain ("ยังไม่เจอ")       | P0        | กันความมั่นใจผิด                   |
| พิสูจน์ตัวตนด้วยสิ่งที่ฟุตเทจเช็คได้     | P0        | จุดขายที่แปลกใหม่สุด               |
| QR claim pass + เจ้าหน้าที่ยืนยันตอนรับ  | P0        | ปิด loop                           |
| reverse video search (ค้นเฟรมที่ของโผล่) | P1        | กู้ของที่ detector พลาด            |
| staff review dashboard (เคสกลาง)         | P1        | HITL                               |
| แจ้งเตือนเมื่อเจอภายหลัง (กรณี abstain)  | P2        | —                                  |
| Face match                               | ❌ ตัดออก | biometric = ความเสี่ยง PDPA สูงสุด |

## 6. ขอบเขตเดโม & เส้นตัด

**ต้องรันสดให้ได้:** แจ้ง → จับคู่ → เลือก candidate → พิสูจน์ → QR pass

**ตัดได้ถ้าไม่ทัน (เรียงลำดับ):** agent/token flow → face match → staff queue อัจฉริยะ (ทำเป็นแค่สถานะ) → live CV detection (ใช้ scenario จัดฉาก + replay แทน)

## 7. Success Metrics

| ระดับ         | ตัวชี้วัด                                                                                     |
| ------------- | --------------------------------------------------------------------------------------------- |
| เดโม          | รัน end-to-end สดจบได้, 3 จุดขายทำงาน, ตอบ Q&A กรรมการได้                                     |
| ผลิตภัณฑ์จริง | recovery rate, เวลาเฉลี่ยจากแจ้ง→เจอ, false-positive rate, สัดส่วนเคสที่ปิดได้โดยไม่ต้องใช้คน |

> หมายเหตุ: เพดาน recall ถูกจำกัดด้วย **physical camera coverage** ไม่ใช่ความเก่งของโมเดล — ต้องสื่อสารตรงๆ ว่า "ครอบคลุมในพื้นที่กล้องเห็น"

## 8. Demo Positioning (16 ก.ค.)

- วางระบบเป็น **"ผู้ช่วยเจ้าหน้าที่"** ไม่ใช่ "ระบบอัตโนมัติเต็มตัว" — จุดอ่อนกลายเป็นความน่าเชื่อถือ
- Edge ยังรันด้วย replay (hardware ยังไม่ถึง) → พูดอย่างซื่อสัตย์ว่า simulate อยู่
- เตรียมสไลด์ 1 หน้าเรื่อง coverage + privacy ไว้ตอบ Q&A

## 9. Non-Goals (สิ่งที่จงใจไม่ทำ)

- ไม่ทำ face recognition / biometric
- ไม่อ้างว่ารู้ "ของข้างในกระเป๋า" จากกล้อง
- ไม่รับประกันว่าเจอของทุกชิ้น (จำกัดด้วยมุมกล้อง)
- ไม่เก็บวิดีโอดิบขึ้น cloud
