# UpFound — Product Specification

ระบบตามหาของหายในพื้นที่ปิด (ห้างสรรพสินค้า สนามบิน อาคารสำนักงาน) ที่ใช้การวิเคราะห์ภาพจากกล้องวงจรปิดเพื่อสร้าง “เส้นทางของวัตถุ” และให้เจ้าของพิสูจน์ความเป็นเจ้าของด้วยหลักฐานที่ระบบตรวจสอบได้ ออกแบบโดยคำนึงถึงความเป็นส่วนตัวเป็นหลัก

---

## 1. บทสรุป

UpFound ช่วยลดภาระงานของเจ้าหน้าที่และเพิ่มอัตราการได้ของคืน โดยเปลี่ยนกระบวนการตามหาของหายจากการพึ่งแรงคนล้วน ๆ มาเป็นกระบวนการที่มีระบบช่วยจับคู่ของกับเจ้าของ พร้อมกลไกพิสูจน์ความเป็นเจ้าของที่น่าเชื่อถือและตรวจสอบย้อนกลับได้

หลักการสำคัญคือ ระบบไม่คาดเดาว่าใครเป็นเจ้าของ แต่ทำหน้าที่รวบรวมหลักฐานที่ตรวจสอบได้ และให้เจ้าหน้าที่เป็นผู้ตัดสินใจขั้นสุดท้าย

## 2. ปัญหาและที่มา

การจัดการของหายในพื้นที่ปิดในปัจจุบันอาศัยแรงคนเป็นหลัก เจ้าหน้าที่ต้องไล่ดูภาพจากกล้องด้วยตนเอง ของที่เก็บได้ถูกกองไว้โดยไม่มีระบบจับคู่กับผู้แจ้ง และผู้ที่ทำของหายต้องสอบถามทีละจุด ส่งผลให้อัตราการได้ของคืนต่ำ ใช้เวลานาน และกระบวนการยืนยันความเป็นเจ้าของขาดความรัดกุม

## 3. กลุ่มผู้ใช้

| กลุ่มผู้ใช้ | ความต้องการหลัก |
|---|---|
| ผู้ที่ทำของหาย | แจ้งได้ง่าย ทราบผลเร็ว และรับของคืนได้สะดวก |
| เจ้าหน้าที่ / ฝ่ายรักษาความปลอดภัย | เครื่องมือช่วยจับคู่ของกับเจ้าของ ลดเวลาในการตรวจสอบภาพ และตัดสินใจส่งมอบได้อย่างมั่นใจ |
| ผู้บริหารพื้นที่ | เพิ่มอัตราการได้ของคืน ปฏิบัติตามกฎหมายคุ้มครองข้อมูลส่วนบุคคล และยกระดับคุณภาพบริการ |

## 4. หลักการออกแบบ

ระบบไม่คาดเดาความเป็นเจ้าของ แต่สร้างเส้นทางของวัตถุที่ตรวจสอบได้ และให้ผู้ใช้ยืนยันด้วยข้อมูลที่ระบบสามารถเทียบกับภาพจากกล้องได้จริง บทบาทของระบบคือผู้ช่วยเจ้าหน้าที่ในการตัดสินใจ ไม่ใช่ผู้ตัดสินใจแทน

แนวทางนี้เป็นสิ่งที่ทำให้ระบบมีความน่าเชื่อถือและสอดคล้องกับหลักความเป็นส่วนตัวไปพร้อมกัน

## 5. การเดินทางของผู้ใช้

**กระบวนการตรวจจับ (ทำงานอัตโนมัติเบื้องหลัง):** เมื่อกล้องตรวจพบวัตถุที่ถูกวางทิ้งไว้นิ่งเกินระยะเวลาที่กำหนด ระบบจะบันทึกเป็นเหตุการณ์เบื้องต้นพร้อมเส้นทางของวัตถุนั้นไว้ เพื่อรอการเชื่อมโยงกับผู้แจ้ง

**กระบวนการตามหา (เริ่มเมื่อมีผู้แจ้ง):**

1. ผู้ใช้แจ้งของหายผ่านแบบฟอร์ม โดยระบุประเภท สี ยี่ห้อ จุดเด่น พื้นที่ และช่วงเวลา (การแนบรูปเป็นทางเลือก ไม่บังคับ)
2. ระบบเสนอรายการที่เป็นไปได้พร้อมภาพจากกล้องและลำดับเวลา
3. ผู้ใช้เลือกวัตถุที่ตรงจากภาพที่ระบบแสดง
4. ผู้ใช้พิสูจน์ความเป็นเจ้าของด้วยคำถามที่อ้างอิงสิ่งที่ปรากฏในกล้อง (เช่น พื้นที่ที่อยู่ จำนวนผู้ที่มาด้วย สีเสื้อ ทิศทางการเดินเข้า) และอาจมีการเทียบใบหน้าแบบเฉพาะกรณีเป็นชั้นเสริม
5. เมื่อผ่านการพิสูจน์ ระบบออกบัตรรับของดิจิทัล (QR)
6. ผู้ใช้ไปยังจุดรับของ เจ้าหน้าที่ตรวจสอบของภายในเพื่อยืนยันก่อนส่งมอบ

## 6. ขอบเขตฟีเจอร์

| ฟีเจอร์ | ระดับความสำคัญ | หมายเหตุ |
|---|---|---|
| แบบฟอร์มแจ้งของหายแบบมีโครงสร้าง | สูง | รองรับกรณีไม่มีรูป |
| จับคู่รายการที่เป็นไปได้ พร้อมภาพและลำดับเวลา | สูง | คุณค่าหลักของระบบ |
| การประเมินความมั่นใจและการปฏิเสธเมื่อไม่ชัดเจน | สูง | ลดการจับคู่ผิดพลาด |
| การพิสูจน์ความเป็นเจ้าของด้วยหลักฐานที่ตรวจสอบได้ | สูง | จุดเด่นด้านความน่าเชื่อถือ |
| บัตรรับของดิจิทัล และการยืนยันโดยเจ้าหน้าที่ขณะส่งมอบ | สูง | ปิดกระบวนการ |
| การค้นย้อนหลังในวิดีโอเพื่อหาเฟรมที่วัตถุปรากฏ | ปานกลาง | ครอบคลุมกรณีที่การตรวจจับอัตโนมัติพลาด |
| หน้าจอตรวจสอบสำหรับเจ้าหน้าที่ (กรณีความมั่นใจปานกลาง) | ปานกลาง | กระบวนการที่มีคนร่วมตัดสินใจ |
| การเทียบใบหน้าแบบเฉพาะกรณี (ephemeral) | ปานกลาง | เทียบแบบหนึ่งต่อหนึ่งภายในกรณีแล้วลบทันที ไม่มีฐานข้อมูลใบหน้า |
| การแจ้งเตือนเมื่อพบวัตถุในภายหลัง | ต่ำ | สำหรับกรณีที่ระบบยังไม่พบในครั้งแรก |
| การจดจำใบหน้า (ฐานข้อมูลใบหน้า / การระบุตัวบุคคล) | ไม่อยู่ในขอบเขต | ข้อมูลชีวมิติแบบจัดเก็บถาวรมีความเสี่ยงด้านความเป็นส่วนตัวสูง |

## 7. ตัวชี้วัดความสำเร็จ

- อัตราการได้ของคืน (recovery rate)
- เวลาเฉลี่ยตั้งแต่แจ้งจนพบรายการที่ตรง
- อัตราการจับคู่ผิดพลาด (false-positive rate)
- สัดส่วนกรณีที่ปิดได้โดยใช้การแทรกแซงของเจ้าหน้าที่น้อยที่สุด
- ความพึงพอใจของผู้ใช้และเจ้าหน้าที่

ทั้งนี้ ความครอบคลุมของระบบถูกจำกัดด้วยขอบเขตการมองเห็นของกล้องเป็นหลัก ระบบจึงให้บริการ “ภายในพื้นที่ที่กล้องครอบคลุม”

## 8. สิ่งที่ไม่อยู่ในขอบเขต

- ไม่ทำการจดจำใบหน้า ไม่มีฐานข้อมูลใบหน้า และไม่ระบุตัวบุคคล (การเทียบใบหน้าแบบหนึ่งต่อหนึ่งเฉพาะกรณีแล้วลบทันที ไม่ถือเป็นฐานข้อมูลใบหน้า)
- ไม่อ้างความสามารถในการระบุสิ่งของภายในกระเป๋าจากภาพกล้อง
- ไม่รับประกันการพบวัตถุทุกชิ้น เนื่องจากถูกจำกัดด้วยมุมมองของกล้อง
- ไม่จัดเก็บวิดีโอต้นฉบับขึ้นระบบคลาวด์

## 9. ประเด็นที่ต้องตัดสินใจ

- กรณีของมีมูลค่าสูงหรือเอกสารสำคัญ ควรมีขั้นตอนการพิสูจน์ที่เข้มงวดกว่ามาตรฐานหรือไม่
- นโยบายความรับผิดชอบกรณีระบบจับคู่ผิดและมีการส่งมอบของผิดเจ้าของ
- ระยะเวลาการเก็บรักษาข้อมูลที่เหมาะสมและเป็นที่ยอมรับขององค์กรผู้ใช้งาน
